# opendata-motomachi-card

> 日本語のREADMEはこちらです: [README.ja.md](README.ja.md)

This repository provides open data related to the historical Motomachi Water Distribution Plant in Hakodate, Japan.

Established in 1889, the Motomachi plant was the second modern waterworks in Japan, following Yokohama. It was the first to be designed and supervised entirely by Japanese engineers. The "Naka-ku Reservoir," built at that time, is the oldest active reservoir in Japan, having served for over 120 years.

> 元町配水場は、明治２２年(1889)、横浜に次いで日本で2番目の近代水道として誕生しました。日本人が初めて設計・監督し、この時に建設された「中区配水地」は、120年以上の歳月を経過した現在もその役割を果たす現役では日本最古の配水地です。

## Demo

- **[元町配水場WebCard一覧 (Motomachi Water Distribution Plant WebCard Gallery)](https://code4fukui.github.io/opendata-motomachi-card/)**

The demo site displays a gallery of historical "WebCards." Each card features a photograph, a title, a detailed description, and related categories from the plant's history.

## Data

### Main Data File
- **[motomachi-card.csv](motomachi-card.csv)**

A comprehensive CSV file containing information about each historical card. This file is generated by combining data from the original `cards.json` and `cardsets.json` files. It links each card to its corresponding photograph, description, and thematic sets.

### Photos
All associated historical photographs are downloaded into the `photo/` directory, named by their `photo_id` (e.g., `01-01.jpg`).

## How to Regenerate the Data

Deno is required to run the data processing scripts.

1.  **Download the source JSON files:**
    ```bash
    # Fetches cards.json, photos.json, and cardsets.json into the data/ directory
    deno run --allow-net --allow-write download_json.js
    ```

2.  **Download all associated photos:**
    ```bash
    # Downloads images listed in data/photos.json into the photo/ directory
    deno run --allow-net --allow-read --allow-write download_photos.js
    ```

3.  **Generate the final CSV file:**
    ```bash
    # Combines data/cards.json and data/cardsets.json into motomachi-card.csv
    deno run --allow-read --allow-write make_csv.js
    ```
